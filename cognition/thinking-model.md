# Thinking Model（思考モデル）

## STEP 1: 問題定義

- 何を実現するのかを1文で定義する
- 成功条件を計測可能な形で定義する
- `PROJECT.md` のスコープ内か確認する

## STEP 2: 制約確認

- WordPress 標準 API で実現可能か
- 既存ファイル・既存責務に収まるか
- セキュリティ / A11y / 性能の最低条件を満たせるか

## STEP 3: 解決案作成（最低3案）

- A案: WordPress ネイティブ中心
- B案: Block API / Interactivity API 中心
- C案: 独自実装（必要時のみ）

## STEP 4: 比較評価

評価軸（各 1〜5 点）:

1. WordPress 自然度
2. 単純性
3. 保守性
4. パフォーマンス
5. アクセシビリティ

## STEP 5: 実装方針確定

- 最高得点案を採用
- 最小実装単位を定義
- 先にテスト観点とレビュー観点を確定
