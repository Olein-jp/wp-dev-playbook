# PROJECT.md 記入例

このファイルは記入見本です。実運用時は `PROJECT.md` に転記してください。

## 1. 基本情報

- プロダクト種別: `plugin`
- プロジェクト名: Editorial Checklist Assistant
- 一言説明: 投稿編集画面で公開前チェックを支援する管理者向けプラグイン
- 提供価値（解決する課題）: 公開ミス（画像 alt 未設定、見出し構造崩れ、抜けリンク）を事前に減らす

## 2. 対象と範囲

- 想定ユーザー: 編集者、コンテンツ管理者
- 利用シーン: 投稿公開前の最終確認
- スコープ内:
  - 投稿保存時のチェック結果表示
  - サイドバーにチェックリスト表示
  - 管理画面のみで動作
- スコープ外（やらないこと）:
  - 自動修正
  - フロント表示への介入
  - 外部サービス連携

## 3. 主な機能

- 機能1: 投稿内容を解析し、チェック項目ごとに Pass/Fail を表示
- 機能2: Fail 項目の理由を編集者へ提示
- 機能3: 最終保存時に警告を表示（公開はブロックしない）

## 4. 技術要件

- 最低 WordPress バージョン: 6.5
- 最低 PHP バージョン: 8.1
- 必須依存ライブラリ（理由つき）: なし（Core API と `@wordpress/*` のみ使用）
- 対応ブラウザ方針: WordPress 管理画面サポート範囲に準拠
- 多言語対応方針（i18n/l10n）: 文字列はすべて翻訳関数を使用

## 5. 非機能要件

- パフォーマンス目標（例: 初期表示、クエリ数）: 追加 SQL クエリを 0 に保つ
- アクセシビリティ目標（例: WCAG レベル）: WCAG 2.1 AA 相当
- セキュリティ要件（例: 権限、nonce、監査ログ）: `edit_posts` 以上のみ利用可、状態変更時は nonce 検証
- 保守性要件（例: 責務分離、命名規則）: UI とチェックロジックを分離、関数名に責務を明記

## 6. データと互換性

- 保存データ（options/post meta/custom table 等）: post meta（最新チェック日時のみ）
- 既存データ移行の有無: なし
- 後方互換ポリシー: 既存投稿に影響しない追加型リリースのみ

## 7. 受け入れ条件（Acceptance Criteria）

- [ ] 投稿保存時、チェック結果が 3 項目以上表示される
- [ ] `edit_posts` 未満ユーザーには機能が表示されない
- [ ] 既存の投稿保存フローを阻害しない

## 8. リリース運用

- バージョニング方針: SemVer（互換破壊はメジャー）
- 変更履歴の管理方法: `CHANGELOG.md` を Keep a Changelog 形式で管理
- ロールバック方針: 前バージョン ZIP へ即時復帰可能にする

## 9. 未確定事項

- 要確認1: チェック項目の優先度表示ルール
- 要確認2: マルチサイトでの有効化単位（ネットワーク/サイト別）
